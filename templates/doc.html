get /api/declarations

Obtenir toutes les informations des déclarations de la ville déclarées entre deux dates.

  • Request
  • Response

Query Parameters

  • du: required(date-only)

    Date de début de la recherche des déclarations.

    Example:

    2015-05-20
  • au: required(date-only)

    Date de fin de la recherche des déclarations.

    Example:

    2016-01-02

HTTP status code 200

Body

Media type: application/json

Type: array of object

Items: DeclarationVilleExistante

  • id: required(number)
  • no_declaration: required(string)
  • date_declaration: required(string)
  • date_insp_vispre: required(string)
  • nbr_extermin: required(number)
  • date_debut_trait: required(string)
  • date_fin_trait: required(string)
  • no_qr: required(string)
  • nom_arr: required(string)
  • coord_x: required(number)
  • coord_y: required(number)
  • longitude: required(number)
  • latitude: required(number)

delete /api/declarations

Supprimer toutes les déclarations de la ville déclarées entre deux dates et provenant d'un quartier spécifié.

  • Request
  • Response

Query Parameters

  • du: required(date-only)

    Date de début de la recherche des déclarations.

    Example:

    2015-05-20
  • au: required(date-only)

    Date de fin de la recherche des déclarations.

    Example:

    2016-01-02
  • quartier: required(string)

    Nom du quartier dans lequel il faut supprimer les déclarations.

    Example:

    Est

HTTP status code 200

Les déclarations ont été supprimées avec succès.

HTTP status code 404

Ils n'existent pas de déclarations pour ce quartier entre ces dates.

  • Response

Media type: application/json

Type: object

Properties

/api/user post

post

Ajouter un nouvel utilisateur.

post /api/user

Ajouter un nouvel utilisateur.

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • utilisateur: required(string)
  • courriel: required(string)
  • quartiers_a_surveiller: required(string)

    Une liste des quartiers à surveillés séparés par des virgules.

  • mot_de_passe: required(string)

Example:

{
  "utilisateur": "Samuel de Champlain",
  "courriel": "samuel_dechamplain@hotmail.com",
  "quartiers_a_surveiller": "Beaurivage,Est",
  "mot_de_passe": "jaimelesponts!"
}

HTTP status code 201

L'utilisateur a été créé avec succès.

Body

Media type: application/json

Type: object

Properties
  • id: required(number)
  • utilisateur: required(string)
  • hashed_password: required(string)

    Le mot de passe de l'utilisateur qui a été haché avec son salt.

  • courriel: required(string)
  • quartiers_a_surveiller: required(string)
  • salt: required(string)

    Une chaîne de caractère aléatoire générée par le programme.

  • pic_id: required(string)

    L'identifiant pour la photo ajoutée par l'utilisateur. Initialement null lors de la création de l'utilisateur.

Example:

{
  "id": 2,
  "utilisateur": "Bob Lebricoleur",
  "hashed_password": "021937u0438hbngdsakjn",
  "courriel": "bobby@lebricoleur.ca",
  "quartiers_a_surveiller": "Est,Anjou",
  "salt": "90783r6y7btfadc7adsc",
  "pic_id": null
}
